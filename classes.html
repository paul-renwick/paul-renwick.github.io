What are the best practices associated with using classes vs. ids?

In order to understand what are considered best practices associated with classes and ID’s, it is important to first understand the primary differences between the two.

So what are the differences? ID’s are unique. Each page is only able to have one element with an ID. Classes are not unique, a web page can have a class used on multiple elements, you can also have multiple classes on the same element.

And how are they similar? They both serve as ‘hooks’ from which properties can be assigned to an element by giving the class or ID values in css.

Best practices regarding the two involve not indulging in superfluous classes or IDs. If an element does not require a class to serve it’s intended purpose, then there is no need to add on unnecessary classes or an ID. If you are writing a style that you want applied to a bunch of different elements across the page, then it is best to assign them to a class, so that any changes to the style will be automatically assigned to all relative elements on the website. Alternatively, if only one element is going to be effected by a particular style, then this would be a good time to assign it an ID.
